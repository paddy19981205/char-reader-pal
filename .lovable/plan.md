
# 強化 OCR 提示詞：減少幻覺 + 提高辨識準確率

## 問題分析

目前的提示詞雖然已加入「禁止替換」的指令，但模型仍可能產生幻覺。原因：
1. 提示詞結構上，「禁令」放在後面，模型可能在辨識過程中已經「腦補」了
2. 缺少具體的辨識技巧引導（如何處理看不清的字）
3. 校對步驟第一條仍然要求「根據上下文語意修正」，與後面的「不要替換」矛盾
4. 沒有要求模型標註不確定的字，導致模型傾向直接猜測

## 改進方案

修改 `supabase/functions/ocr/index.ts` 的 OCR 和校對提示詞：

### 1. OCR 提示詞重構

- 將「忠實辨識」原則提到最前面（System-level 優先級）
- 新增「遇到看不清的字」的處理策略：仔細看筆畫結構，不要靠猜測
- 加入 temperature=0 參數降低隨機性
- 強化逐欄辨識的具體方法：要求模型描述每個字的筆畫特徵再決定是什麼字
- 移除可能誤導模型的「常見易混淆字提醒」（這反而提示模型去做替換）

### 2. 校對提示詞修正

- 移除第一條「根據上下文語意修正」——這與防幻覺目標直接衝突
- 校對範圍嚴格限定為：只修正「單字筆畫極度相似且明顯寫錯」的情況（如 已/己、土/士）
- 加入具體禁令範例

### 3. 加入 temperature=0

在兩個 AI 呼叫中都設定 `temperature: 0`，減少模型的創造性輸出。

## 技術細節

### 修改檔案
- `supabase/functions/ocr/index.ts`

### OCR 提示詞變更

重構為以下結構（核心原則放最前面）：

```
你是一個頂尖的繁體中文手寫文字辨識專家。

【最高原則】你的唯一任務是「看圖辨字」，忠實記錄圖片中每個方格裡的手寫文字。
- 你是一台掃描機，不是編輯，不是校對，不是作文老師
- 絕對不要根據語意、常識、成語知識來「修正」或「替換」你看到的文字
- 如果你看到的筆畫組合成「移山為平地」，就輸出「移山為平地」，即使你知道常見說法是「積水成淵」
- 寧可輸出一個看起來不通順的字，也不要自作主張替換

這張圖片是一張中文稿紙（方格紙/原稿用紙）。

背景資訊：
- 台灣學生的手寫作文，繁體中文
- 直書（縱書）格式：從上到下、從右到左
- 每格一字或一個標點符號
- 稿紙分成數個區塊

辨識步驟：

第一步：觀察稿紙結構（在心中完成，不要輸出）
- 數一數有幾個區塊、每個區塊幾欄
- 確認哪些欄有文字

第二步：逐欄、逐格辨識
- 從最右邊區塊的最右欄開始，由上往下
- 對每個字：仔細觀察筆畫的每一劃，根據筆畫形狀判斷是什麼字
- 不確定的字：放大觀察筆畫細節，根據你看到的筆畫做出最佳判斷，但絕不要用上下文語意來猜
- 讀完一欄移到左邊一欄，一個區塊讀完移到左邊區塊

第三步：逐字對照圖片確認
- 逐字回頭對照圖片，確認輸出的每個字都與圖片中的筆畫一致
- 如果發現某個字是你「根據語意猜的」而非「根據筆畫看到的」，改回筆畫實際顯示的字

標點符號：。，、？！：；「」『』
分段：欄首空格表示新段落，新段落前空一行，段首加兩個全形空格縮排

請只輸出辨識出的純文字，不要加任何說明。
```

### 校對提示詞變更

```
你是一個繁體中文校對員。以下是 OCR 辨識出的手寫文字。

你的任務非常有限：
1. 只修正「筆畫極度相似的形近字」誤判（例：已/己、土/士、大/太、末/未、日/目、人/入）
2. 修正明顯的標點符號錯誤
3. 保持原始段落結構不變

嚴格禁止：
- 不要根據語意或上下文替換任何詞語
- 不要把不常見的用詞改成常見的用詞
- 不要替換成語、典故或慣用語
- 不要潤飾、增刪任何內容
- 如果原文是「移山為平地」，保留原樣，不要改成任何其他說法

你的角色是修正「看錯一劃」的錯誤，不是修正用詞。

原始辨識文字：
${rawText}

請直接輸出校對後的文字，不要加任何解釋。
```
